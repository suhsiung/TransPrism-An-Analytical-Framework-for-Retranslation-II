## 1. 系統簡介

本系統是針對**重譯文本（retranslation）語言特徵**的自動化分析平台，採**雙 Agent 架構**：

| Agent | 模型 | 職責 |
|-------|------|------|
| **主 Agent（LLM1）** | `google/gemma-4-31b-it` | 研究情境引導、最終研究報告撰寫 |
| **工作程序 Agent（LLM2）** | `google/gemma-4-26b-a4b-it` | 上傳引導、格式驗證 |

系統會對上傳的中文譯本計算 **75 項語言指標**，進行**敘述統計、差異性檢定、機器學習分群與分類**，最後由 LLM 產出一份研究報告。

## 2. 系統需求與啟動

### 環境需求
- Python 3.x
- 套件：`gradio`、`pandas`、`numpy`、`scikit-learn`、`matplotlib`、`python-docx`
- OpenRouter API 金鑰（供 LLM 呼叫）

### 環境變數設定
LLM 透過 OpenRouter 呼叫，需設定以下環境變數：

```
LLM1_API_KEY   你的 OpenRouter 金鑰
LLM1_MODEL     google/gemma-4-31b-it（預設）
LLM2_API_KEY   你的 OpenRouter 金鑰
LLM2_MODEL     google/gemma-4-26b-a4b-it（預設）
```

### 啟動系統

```powershell
cd D:\D11-AIAgentForTranslation_main_v2
python app_agent.py
```

啟動後開啟瀏覽器前往：**http://localhost:7860**

> 💡 修改程式碼後，務必**完全關閉舊的 Python 程序**再重啟，否則舊程序可能仍佔用 7860 埠導致看到舊版畫面。

## 4. 操作流程（八大步驟）

右欄「執行進度」會即時反映目前所在步驟：

| 步驟 | 名稱 | 說明 |
|------|------|------|
| **1** | 情境選擇 | 在對話視窗輸入 `1`、`2`、`3` 或 `4` 選擇研究情境 |
| **2** | 上傳引導 | 系統說明檔名規範與斷詞格式要求 |
| **3** | 文本上傳 | 點選上傳卡片，選擇已斷詞的 `.txt` 檔（可多選）→ 確認上傳 |
| **4** | 格式驗證 | 系統自動檢查是否為 ckip_segmentation_sumin 斷詞格式 |
| **5** | 指標計算 | 計算 75 項語言指標 |
| **6** | 統計分析 | 敘述統計、差異檢定、機器學習 |
| **7** | 報告撰寫 | 主 Agent（LLM1）撰寫研究報告 |
| **8** | 下載成果 | 於各結果分頁檢視與下載 |

> **重要新增步驟**：在步驟 4（格式驗證通過）與步驟 5（指標計算）之間，系統會跳出**「語料 Metadata 說明」表單**，請填寫後按「確認並開始分析」才會進入計算（詳見 [第 7 節](#7-語料-metadata-說明)）。

## 5. 四種研究情境

| 編號 | 情境 | 群組數 | 分析焦點 |
|------|------|--------|----------|
| **1** | 同一原文的人工翻譯 vs 機器翻譯 | 二群（A/B） | 人機差異的語言特徵 |
| **2** | 全譯 vs 編譯（節譯、改寫） | 二群（A/B） | 文本完整性與改寫策略 |
| **3** | 不同譯者的同期譯本 | 二群以上（A/B/C…） | 個人風格差異（排除時代因素） |
| **4** | 同一原文的不同時期人工譯本 | 二群以上（A/B/C…） | 歷時變化、語言現代化、翻譯規範演變 |

> 情境 **1、2 固定為二群**，故 Metadata 表單中的「比較結構」欄位會自動隱藏；情境 **3、4** 則會顯示，可指定群組數量與比較方式。

## 6. 檔案格式要求

### 6.1 檔名規範（決定分組）
- 系統以**檔名第一個字母**作為分組依據。
- 二群：`A_xx.txt`、`B_xx.txt`
- 多群（情境 3、4）：`A_xx.txt`、`B_xx.txt`、`C_xx.txt`…
- 範例：`A_CH1_seg.txt`、`B_DB1_seg.txt`（A 群、B 群各自的個體檔）

### 6.2 斷詞要求（必要條件）
- 文本**必須先經由 [ckip_segmentation_sumin](https://github.com/suhsiung/ckip_segmentation_sumin) 工具完成斷詞**才能上傳。
- 系統會自動驗證格式，不符者會要求重新處理。

### 6.3 內容格式
每行為下列之一：
- `詞_詞性`（例如：`翻譯_VC`）
- `_詞性`
- 空白行（段落分隔）

## 7. 語料 Metadata 說明

為讓系統與研究報告了解語料來源、範圍與比較結構，**上傳並通過格式驗證後、正式分析前**，會跳出 Metadata 說明表單。內容分三層次：

### 一、原文資訊
原文標題、作者、語言、出版年、版本。

### 二、比較結構（僅情境 3、4）
- **群組數量**：二群／三群
- **比較方式**：群組間比較／個體間比較
- （情境 1、2 固定二群，此區自動隱藏）

### 三、群組與個體資訊
- **群組層**（可編輯表格，依檔名首字母自動列出各群）：群組標籤、翻譯類型、翻譯方向、譯者背景、目標讀者。
- **個體層**（可編輯表格，依上傳檔案自動逐筆列出）：譯者／系統名稱、出版年、譯者母語、後編輯程度、重譯序位。

填妥後按 **「確認並開始分析」**；若不需填寫可按 **「略過」** 直接分析。所填內容會編譯成結構化說明，作為 LLM 撰寫研究報告的背景脈絡。

## 8. 結果分頁說明

分析完成後，頂部分頁自動顯示：

### 📊 語言指標分析
- 75 項語言指標的計算結果表格。
- 可下載 CSV / XLSX 原始數據。

### 📈 統計比較分析（含 5 個子分頁）
| 子分頁 | 內容 |
|--------|------|
| **1. 敘述統計** | 75 項指標的平均數、中位數、標準差、最小／最大值、變異係數 |
| **2. 差異性分析** | 群組間差異檢定（t 檢定／ANOVA）、p 值、顯著性、效果量 + 顯著指標箱形圖 |
| **3. 監督式學習** | 隨機森林分類準確率、特徵重要性 Top 20 |
| **4. 非監督式學習** | 層次分群樹狀圖、K-means Elbow、PCA 2D 視覺化、分群結果 |
| **5. 其他** | 下載統計輸出檔案 |

### 📝 LLM輔助分析
- 主 Agent（LLM1）撰寫的深度研究報告，可下載 Word（.docx）與純文字（.txt）。

## 10. 統計方法說明

- **分組依據**：檔名第一個字母（A、B、C…）。
- **差異檢定**：
  - 二群 → Independent t-test（Welch）
  - 三群以上 → One-way ANOVA
  - 顯著水準 α = **0.05**
- **效果量**：t 檢定用 Cohen's d；ANOVA 用 η²（含大／中／小／極小判讀）。
- **監督式學習**：隨機森林分類器，輸出準確率與特徵重要性。
- **非監督式學習**：階層式分群（Ward）、K-means（含 Elbow）、PCA 2D 降維視覺化。

## 11. 常見問題與疑難排解

**Q1：上傳後畫面卡住、無法點選？**
上傳卡片為行內卡片，按「確認上傳」或「略過」後會自動關閉。若仍卡住，請按 `Ctrl+Shift+R` 強制重整。

**Q2：修改設定後畫面沒更新？**
請完全關閉舊 Python 程序再重啟，並用 `Ctrl+Shift+R` 清除瀏覽器快取。確認 7860 埠的程序是新啟動的實例。

**Q3：指標數量不一致 / 顯示舊結果？**
結果分頁的數據在「執行計算」當下才生成。重啟 app 或重整頁面**不會**重算既有結果——需**完整重跑一次分析**（重新上傳→計算）。

**Q4：埠 7860 被佔用？**
```powershell
Get-Process python | Stop-Process -Force
```
確認 7860 完全釋放後再啟動。

**Q5：格式驗證一直失敗？**
確認檔案確實經由 ckip_segmentation_sumin 斷詞，且內容為 `詞_詞性`／`_詞性`／空白行格式。
